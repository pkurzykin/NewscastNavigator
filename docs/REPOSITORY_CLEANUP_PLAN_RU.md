# План cleanup репозитория

Дата актуализации: 2026-03-15

## Зачем нужен этот документ

Документ фиксирует, как репозиторий был доведен до `web-only` состояния без потери данных и deploy-воспроизводимости.

## Целевое состояние

В `main` должно остаться только то, что относится к актуальной web-версии:
- `backend/`
- `frontend/`
- `deploy/`
- `docs/`

Legacy не должен жить в рабочем дереве. Исторический контекст хранится в git history и server backups.

## Что уже сделано

- Добавлен корневой `README.md`.
- Добавлен `.gitignore` под будущий GitHub-репозиторий.
- Уточнены `backend/README.md` и `frontend/README.md`.
- Проведен аудит домашнего сервера и сделан clean deploy-path `/opt/newscast-web`.
- Legacy-данные импортированы в новый web-контур.
- Выполнен production cutover на новый web.
- Установлен `systemd` для production compose.
- Удалены old server runtime и старые server directories legacy/dev-контура.
- Legacy Streamlit-код и старый docs archive удалены из `main`.
- Build-артефакты и временные server/dev хвосты вычищены из рабочего дерева.

## Критерии завершения

- любой новый участник видит только web-контур;
- deploy читается из `deploy/`, а не из ручных server-правок;
- legacy не мешает разработке и не создает ложных инструкций запуска;
- старые данные и rollback-материалы сохранены отдельно от рабочего дерева.

## Что остается делать дальше

Cleanup как миграционный этап закрыт.

Дальше фокус уже другой:
1. развивать web-продукт;
2. держать документацию и deploy-скрипты в актуальном состоянии;
3. не возвращать legacy-артефакты в `main`;
4. проводить любые server-изменения через репозиторий и воспроизводимые runbook'и.
