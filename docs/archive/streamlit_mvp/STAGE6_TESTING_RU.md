# Этап 6 — Экспорт и печать: как тестировать и проверять

Дата: 2026-02-15

## Что реализовано на этапе
- Экспорт проекта в Word (`.docx`).
- Экспорт проекта в PDF (`.pdf`).
- Печатная версия в интерфейсе (`Печатная версия`).
- Экспорт включает:
  - метаданные проекта,
  - строки сценария (блоки, текст, титры, тайминги, комментарии).

## 0. Подготовка зависимостей
Если кнопки экспорта неактивны, установите пакеты в ваш `.venv`:

```bash
cd /Volumes/work/Projects/NewscastNavigator
source .venv/bin/activate
pip install python-docx reportlab
```

## 1. Запуск приложения

```bash
cd /Volumes/work/Projects/NewscastNavigator
source .venv/bin/activate
python3 scripts/migrate_db.py
streamlit run app.py
```

Откройте `http://localhost:8501`.

## 2. Подготовка тестового проекта
1. Откройте любой проект.
2. Убедитесь, что заполнены:
  - метаданные (название, рубрика, хронометраж),
  - несколько строк сценария разных типов,
  - тайминги и титры.

## 3. Проверка Export Word
1. Нажмите `Export Word`.
2. Сохраните файл.
3. Откройте файл в Word/LibreOffice.

Ожидается:
- документ открывается без ошибок,
- метаданные проекта присутствуют,
- таблица со строками сценария корректно заполнена.

## 4. Проверка Export PDF
1. Нажмите `Export PDF`.
2. Откройте файл.

Ожидается:
- PDF открывается,
- текст читаемый,
- есть метаданные и содержимое сценария.

Примечание: для корректной кириллицы в PDF нужен доступный TTF-шрифт на системе (код пытается подобрать автоматически).

## 5. Проверка печатной версии
1. Включите чекбокс `Печатная версия`.
2. Проверьте, что появилась упрощенная табличная раскладка.
3. Используйте печать браузера (`Cmd+P` / `Ctrl+P`) и сохраните как PDF.

Ожидается:
- печатная раскладка аккуратная,
- структура соответствует проекту.
- в предпросмотре печати отображается только блок печатной версии (а не вся страница Streamlit).

## 6. Negative tests
1. Попробуйте экспортировать проект с пустыми полями.
2. Проверьте, что экспорт все равно формируется (пустые значения отображаются как пусто/`-`).

## 7. Regression тест
Проверьте, что не сломалось:
- `MAIN`,
- `ARHIV`,
- редактирование строк,
- загрузка/удаление файлов (Этап 5).

## 8. Критерии приемки Этапа 6
Этап считается принятым, если:
- Word экспорт работает,
- PDF экспорт работает,
- печатная версия отображается,
- экспорт отражает актуальные данные проекта,
- предыдущий функционал не деградировал.
