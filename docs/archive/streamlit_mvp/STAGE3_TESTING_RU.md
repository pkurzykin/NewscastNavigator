# Этап 3 — text-editor: как тестировать и проверять (подробно)

Дата: 2026-02-15

## Что должно быть реализовано на этом этапе
- Верхняя зона `text-editor` с метаданными проекта.
- Режим титрования (`Режим титрования`) для упрощенного отображения.
- Построчное редактирование сценария с полями:
  - тип блока,
  - текст,
  - титры/ФИО,
  - имя файла,
  - `TC IN`,
  - `TC OUT`,
  - другой комментарий.
- Валидация таймингов:
  - формат `MM:SS` или `HH:MM:SS`,
  - `TC OUT` не меньше `TC IN`.
- Комментарии к каждой строке сценария.

## 0. Подготовка

```bash
cd /Volumes/work/Projects/NewscastNavigator

python3 scripts/migrate_db.py
streamlit run app.py
```

Откройте `http://localhost:8501`.

## 1. Smoke test интерфейса text-editor
1. Войдите в приложение.
2. Выберите проект в MAIN.
3. Прокрутите до блока `Text-Editor (Этап 3)`.

Должны быть видны:
- верхние карточки/блоки метаданных,
- кнопка/чекбокс `Режим титрования`,
- таблица строк сценария с колонками.

## 2. Проверка метаданных проекта
1. В блоке `Метаданные проекта` измените:
  - название,
  - рубрику,
  - хронометраж,
  - путь к файлам.
2. Нажмите `Сохранить метаданные`.
3. Убедитесь, что изменения отображаются сразу.
4. Обновите страницу браузера (F5) и проверьте, что данные сохранились.

Проверка статуса:
1. Измените статус проекта (если роль позволяет).
2. Сохраните метаданные.
3. Убедитесь, что статус в MAIN и text-editor обновился.

## 3. Проверка назначений ролей
1. Измените автора/исполнителя/корректора в форме метаданных.
2. Нажмите `Сохранить метаданные`.
3. Проверьте, что новые значения отображаются в верхнем блоке и в таблице MAIN.

## 4. Проверка редактирования строк сценария
1. Для одной строки измените:
  - тип блока,
  - текст,
  - титры/ФИО,
  - имя файла,
  - тайминги.
2. Нажмите `Сохранить строку`.
3. Убедитесь в сообщении об успешном сохранении.
4. Обновите страницу и проверьте, что значения сохранены.

## 5. Проверка валидации таймингов

### 5.1 Неверный формат
1. В `TC IN` введите, например, `abc`.
2. Нажмите `Сохранить строку`.
3. Ожидается ошибка формата.

### 5.2 Конец меньше начала
1. Введите `TC IN = 01:10`, `TC OUT = 00:30`.
2. Нажмите `Сохранить строку`.
3. Ожидается ошибка: `TC OUT не может быть меньше TC IN`.

### 5.3 Корректный формат
1. Введите `TC IN = 00:38`, `TC OUT = 00:56`.
2. Нажмите `Сохранить строку`.
3. Ожидается успешное сохранение.

## 6. Проверка режима титрования
1. Включите `Режим титрования`.
2. Убедитесь, что таблица упростилась (меньше колонок, фокус на тексте/титрах).
3. Сохраните строку в этом режиме.
4. Отключите режим и убедитесь, что данные не потерялись.

## 7. Проверка комментариев по строкам
1. У конкретной строки введите комментарий.
2. Нажмите `Сохранить комментарий`.
3. Проверьте, что комментарий отображается в списке комментариев этой строки.
4. Добавьте комментарий под другим пользователем и убедитесь, что виден автор.

## 8. Проверка добавления новых строк
1. Внизу выберите тип нового блока.
2. Нажмите `Добавить строку`.
3. Убедитесь, что строка появилась в таблице.
4. Заполните ее и сохраните.

## 9. Regression тест
Проверьте, что не сломалось из предыдущих этапов:
- вход в систему,
- фильтры MAIN,
- 3 способа создания проекта,
- архивирование с подтверждением.

## 10. Критерии приемки Этапа 3
Этап считается принятым, если:
- text-editor отображает и сохраняет метаданные проекта,
- строки сценария редактируются по колонкам,
- валидация таймингов работает корректно,
- режим титрования работает,
- комментарии по строкам работают,
- функционал Этапа 2 не деградировал.
