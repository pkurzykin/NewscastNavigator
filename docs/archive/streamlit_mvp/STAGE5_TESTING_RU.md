# Этап 5 — Локальное файловое хранилище: как тестировать и проверять

Дата: 2026-02-15

## Что должно быть реализовано на этапе
- Загрузка файла к конкретной строке сценария (`text-editor`).
- Хранение файла на локальном диске сервера (без облаков).
- Создание записи о файле в БД (`project_files`).
- Валидация:
  - разрешенные расширения,
  - ограничение размера файла.
- Удаление файла из строки (и из БД, и с диска).
- Базовый backup БД и папки `storage`.

## 0. Подготовка

```bash
cd /Volumes/work/Projects/NewscastNavigator
source .venv/bin/activate
python3 scripts/migrate_db.py
streamlit run app.py
```

Откройте `http://localhost:8501`.

## 1. Smoke test: секция файлов в text-editor
1. Войдите и откройте любой проект в `MAIN`.
2. Прокрутите к строкам сценария.
3. Убедитесь, что под каждой строкой есть блок:
  - список файлов строки,
  - загрузка файла,
  - кнопка сохранения файла.

## 2. Проверка загрузки файла
1. Выберите строку сценария.
2. Загрузите небольшой тестовый файл допустимого типа (например `.txt` или `.pdf`).
3. Нажмите `Сохранить файл`.

Ожидается:
- сообщение об успешной загрузке,
- файл появился в списке файлов строки,
- отображается путь файла на диске.

## 3. Проверка записи в БД
В отдельном терминале:

```bash
cd /Volumes/work/Projects/NewscastNavigator
python3 - <<'PY'
import sqlite3
from pathlib import Path
db = Path("data/app.db")
conn = sqlite3.connect(db)
rows = conn.execute("SELECT id, project_id, element_id, original_name, storage_path, file_size FROM project_files ORDER BY id DESC LIMIT 5").fetchall()
conn.close()
print(rows)
PY
```

Ожидается: в таблице `project_files` есть запись по загруженному файлу.

## 4. Проверка, что файл реально лежит на диске
Путь виден в интерфейсе. Также можно проверить в терминале:

```bash
ls -la storage/projects
```

Ожидается: внутри появились каталоги проекта и файлы.

Примечание: если вы задали `Путь к файлам проекта` в метаданных, файлы будут сохраняться туда.

## 5. Проверка валидации типа файла
1. Попробуйте загрузить файл с запрещенным расширением.
2. Нажмите `Сохранить файл`.

Ожидается:
- сообщение с ошибкой о недопустимом типе,
- файл не сохраняется.

## 6. Проверка валидации размера
1. Попробуйте загрузить файл больше лимита (по умолчанию 512 MB).
2. Нажмите `Сохранить файл`.

Ожидается:
- сообщение об ограничении размера,
- файл не сохраняется.

## 7. Проверка удаления файла
1. В списке файлов строки нажмите `Удалить`.
2. Убедитесь, что запись исчезла из интерфейса.
3. Повторно проверьте `project_files` в БД.
4. Проверьте, что файла больше нет на диске.

## 8. Проверка базового backup
В терминале:

```bash
cd /Volumes/work/Projects/NewscastNavigator
python3 scripts/backup_local_data.py
```

Ожидается:
- создана папка вида `backups/backup_YYYYMMDD_HHMMSS`,
- внутри есть:
  - `app.db` (если БД существовала),
  - копия `storage` (если папка существовала),
  - `manifest.json`.

## 9. Regression тест
После операций с файлами проверьте:
- `MAIN` работает,
- `ARHIV` работает,
- редактирование строк и комментарии не сломались.

## 10. Критерии приемки Этапа 5
Этап считается принятым, если:
- загрузка и удаление файлов работают,
- файлы физически хранятся локально на сервере,
- записи в `project_files` корректны,
- валидация типа и размера работает,
- базовый backup выполняется успешно.
