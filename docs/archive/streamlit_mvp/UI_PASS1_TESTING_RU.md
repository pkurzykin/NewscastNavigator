# UI Pass 1 — Проверка табличного вида

Дата: 2026-02-15

## Что изменено
- Добавлена визуальная шапка в стиле макета для экранов `MAIN`, `ARHIV`, `LOGIN`.
- Добавлена принудительная светлая тема (чтобы не было светлого текста на светлом фоне).
- `MAIN` и `ARHIV` таблицы получили более «табличный» вид и бейджи статусов.
- `MAIN` и `text-editor` разнесены в разные разделы (`MAIN`, `EDITOR`, `ARHIV`).
- В `MAIN` действия собраны в единый блок кнопок по логике вашего референса.
- В `text-editor` добавлен верхний цветной блок, приближенный к вашему шаблону.

## Как обновить на сервере
На Mac:
```bash
rsync -av \
  --exclude '.venv' \
  --exclude '__pycache__' \
  /Volumes/work/Projects/NewscastNavigator/ \
  wysiati@192.168.2.200:/opt/newscast-navigator/
```

На сервере:
```bash
cd /opt/newscast-navigator
docker compose up -d --build
docker compose ps
```

## Чек-лист проверки
1. Открой `http://192.168.2.200`.
2. Проверь `LOGIN`: желтая широкая шапка с `Newscast Navigator LOGIN`.
3. После входа проверь `MAIN`:
   - желтая шапка `MAIN`;
   - единый блок кнопок действий;
   - таблица с колонками: дата, название, рубрика, статус, автор, исполнитель, корректор;
   - в названии строки есть `#ID`.
4. Нажми `Открыть text-editor` в `MAIN`:
   - должен открыться отдельный раздел `EDITOR`;
   - `MAIN` таблица на этом экране не должна отображаться.
5. Открой `ARHIV` через sidebar или кнопку в `MAIN`:
   - желтая шапка `ARHIV`;
   - действия возврата из архива;
   - таблица архива в едином стиле.
6. В `EDITOR`:
   - есть цветной верхний блок-схема;
   - ниже работают все формы редактирования, сохранения строк, загрузки файлов, экспорт.

## Если UI не обновился
1. Сделай hard refresh в браузере (`Cmd+Shift+R` / `Ctrl+F5`).
2. Перезапусти контейнеры:
```bash
docker compose down
docker compose up -d --build
```
3. Проверь логи:
```bash
docker compose logs app --tail=120
docker compose logs nginx --tail=120
```
