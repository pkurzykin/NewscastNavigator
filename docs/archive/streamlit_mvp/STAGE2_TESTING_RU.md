# Этап 2 — MAIN: как тестировать и проверять (подробно)

Дата: 2026-02-15

## Что должно быть реализовано на этом этапе
- Табличный экран `MAIN` (визуально ближе к вашим таблицам).
- Фильтры списка проектов.
- 3 режима создания проекта:
  - новый пустой,
  - на основе последнего,
  - на основе выбранного.
- Отправка выбранного проекта в архив с подтверждением.

## 0. Подготовка
Работаем из корня проекта:

```bash
cd /Volumes/work/Projects/NewscastNavigator
```

Если используете virtualenv:

```bash
source .venv/bin/activate
```

## 1. Перед тестом — миграции

```bash
python3 scripts/migrate_db.py
```

Ожидается: `Database migrations applied successfully.`

## 2. Запуск приложения

```bash
streamlit run app.py
```

Откройте ссылку из терминала (обычно `http://localhost:8501`).

## 3. Smoke test интерфейса MAIN
После входа проверьте:
- есть заголовок `Newscast Navigator — MAIN`,
- видна таблица проектов,
- есть блок фильтров,
- есть кнопки создания и архивирования.

Если это есть — визуально вы уже в правильном направлении (ближе к вашему макету из `src_doc`).

## 4. Проверка фильтров

### 4.1 Фильтр по статусу
1. Выберите 1 статус.
2. Убедитесь, что в таблице остались только проекты этого статуса.
3. Снимите фильтр и проверьте, что список снова полный.

### 4.2 Фильтр по рубрике
1. Введите часть текста в поле `Рубрика содержит`.
2. Убедитесь, что отображаются только подходящие строки.
3. Очистите поле и проверьте возврат полного списка.

### 4.3 Фильтр по участнику
1. Введите логин пользователя в поле `Участник содержит`.
2. Проверьте, что отфильтровались проекты, где он автор/исполнитель/корректор.

### 4.4 Фильтр по дате
1. Включите `Фильтр по дате создания`.
2. Укажите диапазон дат.
3. Проверьте, что список соответствует диапазону.
4. Проверьте negative-case: дата `от` больше даты `до` -> должна быть ошибка.

## 5. Проверка 3 режимов создания

### 5.1 Создать новый (пустой)
1. Нажмите `Создать новый (пустой)`.
2. Убедитесь, что появился новый проект.
3. Откройте его — внизу должны быть стартовые блоки сценария (`Подводка`, `ЗК`, `Лайф`, `СНХ`).

### 5.2 Создать на основе последнего
1. Нажмите `Создать на основе последнего`.
2. Убедитесь, что появился проект-копия.
3. Проверьте, что элементы сценария скопированы.

### 5.3 Создать на основе выбранного
1. Выберите проект в поле `Открыть проект`.
2. Нажмите `Создать на основе выбранного`.
3. Проверьте, что создана копия именно выбранного проекта.

## 6. Проверка архивирования (с подтверждением)
1. Выберите проект.
2. Нажмите `Отправить выбранный в архив`.
3. Должен появиться блок подтверждения.
4. Нажмите `Отмена` -> проект не архивируется.
5. Повторите и нажмите `Да, архивировать` -> проект уходит в архив.
6. При выключенном `Показывать архив` проект должен исчезнуть из списка.
7. Включите `Показывать архив` -> проект снова виден, но с признаком архива.

## 7. Regression test старого функционала
Проверьте, что не сломалось:
- вход в систему,
- открытие проекта,
- редактирование текста элемента,
- добавление комментария,
- смена статуса (для ролей с правами).

## 8. Критерии приемки Этапа 2
Этап считается принятым, если:
- `MAIN` отображается в виде таблицы,
- все фильтры работают,
- все 3 режима создания работают,
- архивирование работает через подтверждение,
- старый функционал не деградировал.

## 9. Типичные проблемы
- Не видно новые поля/поведение:
  - проверьте, что запускали `python3 scripts/migrate_db.py`.
- Ошибка импорта:
  - запускайте команды из корня проекта.
- Не работает кнопка из-за роли:
  - проверьте, под каким пользователем вы вошли.
