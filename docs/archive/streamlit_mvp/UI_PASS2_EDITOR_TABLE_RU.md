# UI Pass 2 — EDITOR как редактируемая таблица

Дата: 2026-02-16

## Что изменено
- В `EDITOR` строки сценария редактируются в одной таблице, как в Excel.
- Убраны разрозненные формы «по одной строке».
- Сохранение сделано одной кнопкой: `Сохранить таблицу`.
- Добавление строки выполняется кнопкой `Добавить строку` с автоматическим номером `№`.
- Удаление выполняется кликом по строкам прямо в редактируемой таблице, затем кнопкой `Удалить выбранные`.
- Файлы и комментарии переведены на уровень проекта (а не строки).
- При удалении строки из таблицы удаляются и ее связанные комментарии/файлы (из БД; файл на диске тоже очищается по возможности).
- Служебный столбец `ID` скрыт из интерфейса; в таблице виден только пользовательский номер строки `№`.
- Используется одна колонка `Титр`.
- Для строк типа `СНХ` в колонке `Титр` обязательно две строки: `ФИО` на первой, `должность` на второй.

## Как проверить
1. Открой `EDITOR`.
2. Измени несколько ячеек в разных колонках.
3. Для строки с блоком `СНХ` в колонке `Титр` введи:
   - первая строка: ФИО,
   - вторая строка: должность.
4. Нажми `Добавить строку` и проверь, что новая строка получила следующий номер `№`.
5. В редактируемой таблице кликни по одной или нескольким строкам для удаления.
6. Нажми `Удалить выбранные` и проверь, что строки удалились.
7. Нажми `Сохранить таблицу`.
8. Обнови страницу (`Cmd+R`) и проверь, что изменения сохранились.
9. Проверь валидацию:
   - введи неверный `TC IN`/`TC OUT` и попробуй сохранить;
   - введи `TC OUT < TC IN` и проверь ошибку.
10. В панели «Комментарии проекта»:
   - добавь комментарий проекта.
11. В панели «Файлы проекта»:
   - загрузи и удали файл проекта.

## Ожидаемый результат
- Редактирование и структура воспринимаются как единая таблица.
- Нет ощущения «визуал отдельно, формы отдельно».
- Все данные корректно сохраняются после перезагрузки.
