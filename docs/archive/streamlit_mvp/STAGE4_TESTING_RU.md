# Этап 4 — ARHIV: как тестировать и проверять (подробно)

Дата: 2026-02-15

## Что должно быть реализовано на этом этапе
- Отдельный раздел `ARHIV` (через переключатель разделов в боковой панели).
- Таблица архивных проектов.
- Фильтры по названию/рубрике/кто архивировал/дате архивации.
- Возврат проекта из архива в `MAIN` с подтверждением.
- История событий выбранного проекта.

## 0. Подготовка

```bash
cd /Volumes/work/Projects/NewscastNavigator
source .venv/bin/activate
python3 scripts/migrate_db.py
streamlit run app.py
```

Откройте `http://localhost:8501`.

## 1. Smoke test раздела ARHIV
1. Войдите в систему.
2. В боковой панели переключите `Раздел` на `ARHIV`.
3. Проверьте, что открылась страница с заголовком `Newscast Navigator — ARHIV`.

Должны быть видны:
- фильтры архива,
- таблица архивных проектов,
- выбор архивного проекта,
- блок истории событий.

## 2. Подготовка тестовых данных
Если архив пуст:
1. Перейдите в `MAIN`.
2. Архивируйте 2-3 проекта (через кнопку архивирования с подтверждением).
3. Вернитесь в `ARHIV`.

Ожидается: архивные проекты появились в таблице.

## 3. Проверка фильтров ARHIV

### 3.1 Название/тема содержит
1. Введите часть названия.
2. Проверьте, что остаются только подходящие проекты.

### 3.2 Рубрика содержит
1. Введите часть рубрики.
2. Проверьте фильтрацию.

### 3.3 Кто архивировал
1. Введите логин пользователя, который архивировал.
2. Проверьте, что фильтр корректно отрабатывает.

### 3.4 Дата архивации
1. Включите фильтр по дате архивации.
2. Укажите диапазон.
3. Проверьте соответствие списка диапазону.
4. Negative-case: `Дата от > Дата до` -> должна появиться ошибка.

## 4. Проверка возврата проекта в работу
1. Выберите архивный проект.
2. Нажмите `Вернуть выбранный проект в работу`.
3. Появляется подтверждение.
4. Нажмите `Отмена` -> проект остается в архиве.
5. Повторите и нажмите `Да, вернуть в работу`.

Ожидается:
- сообщение об успешном возврате,
- проект исчезает из таблицы архива,
- проект появляется в `MAIN`.

## 5. Проверка прав на возврат
Проверьте под разными ролями:
- `admin` / `editor` — могут вернуть проект.
- остальные роли — видят, что прав недостаточно.

## 6. Проверка истории событий
1. В `ARHIV` выберите проект.
2. В блоке истории проверьте, что есть события:
  - архивирование,
  - изменение статуса,
  - возврат (после восстановления).
3. Проверьте, что видно:
  - время события,
  - пользователь,
  - тип события,
  - old/new значения (где применимо).

## 7. Regression тест
После возврата проекта проверьте:
- `MAIN` работает как раньше,
- проект открывается в `text-editor`,
- данные проекта и строки сценария не потерялись.

## 8. Критерии приемки Этапа 4
Этап считается принятым, если:
- есть отдельный рабочий раздел `ARHIV`,
- фильтры архива работают,
- возврат проекта работает через подтверждение,
- права на возврат соблюдаются,
- история событий отображается,
- предыдущие этапы не деградировали.
